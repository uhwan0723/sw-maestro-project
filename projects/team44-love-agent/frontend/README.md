# Frontend

사용자가 연애 고민 상담 서비스를 실제로 사용하는 화면을 구현하는 영역입니다.

## 담당자

| 담당자 | 역할 | 담당 범위 |
| --- | --- | --- |
| 김준서 | UI 화면 구성 | 레이아웃, 컴포넌트, 화면 전환, 진행 상태, 결과 표시 방식 |
| 박준혁 | 소통 방식 규정화 | 사용자 안내 문구, 버튼 문구, 상태 메시지, 오류/로딩/완료 문구 |

## 작업 기준

- 김준서는 화면의 구조와 사용 흐름을 먼저 잡습니다.
- 박준혁은 같은 화면에서 사용자에게 어떤 문구가 보여야 하는지 정리합니다.
- 화면 구조와 문구는 분리해서 관리하되, 최종 화면에서는 하나의 흐름으로 자연스럽게 보이도록 맞춥니다.
- 에이전트 응답 데이터가 필요하면 김민우에게 필요한 필드 목록을 전달합니다.
- 라운드 이름, 프로토콜 용어, 최종 요약 항목은 신현성의 정의와 맞춥니다.
- 프롬프트와 직접 연결되는 문구는 임지빈과 충돌 여부를 확인합니다.
- 사용자에게 직접 보이는 문구는 [사용자 노출 문구 사전](../docs/team44-love-agent/03_features_workflow/user_messages.md)을 기준으로 적용합니다.

## 예정 화면

- 상담 시작 및 고민 입력 화면
- 슈퍼바이저 질문 분석 화면
- 6개 에이전트 초기 의견 화면
- 라운드별 토론 진행 화면
- 의견 충돌 및 쟁점 정리 화면
- 최종 상담 결과 화면
- 로딩, 오류, 재시도, 완료 상태 화면

## 향후 폴더 배치 기준

실제 프론트엔드 코드가 추가되면 아래 기준으로 나눕니다.

```text
frontend/
├── src/
│   ├── pages/                  # 김준서: 화면 단위 구성
│   ├── components/             # 김준서: 재사용 UI 컴포넌트
│   ├── styles/                 # 김준서: 스타일 및 레이아웃
│   ├── content/                # 박준혁: 화면 문구와 상태 메시지
│   ├── constants/              # 김준서, 박준혁: 단계명, 라벨, 상태값
│   └── types/                  # 김준서, 김민우: API 응답 타입
└── README.md
```

## 완료 기준

- 사용자가 현재 상담 단계가 어디인지 알 수 있어야 합니다.
- 6개 에이전트의 의견과 토론 흐름이 구분되어 보여야 합니다.
- 최종 답변은 슈퍼바이저가 통합한 결과처럼 보여야 합니다.
- 로딩, 오류, 재시도, 완료 상태 문구가 비어 있으면 안 됩니다.
- 화면 문구는 박준혁이 정리한 기준을 따릅니다.
