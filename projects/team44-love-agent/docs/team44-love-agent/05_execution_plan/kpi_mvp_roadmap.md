# 성과 평가 및 실행 계획

## 성공 지표(KPI)

| 구분 | 지표 | 목표 |
| --- | --- | --- |
| 핵심 목표 | Agentic Workflow 완결성 | 사용자 고민 입력부터 최종 요약까지 7개 Agent가 오류 없이 작동하는 비율 100% |
| 핵심 목표 | Agent 간 의견 대립 및 토론 충실도 | 6명의 Agent가 상반되거나 대조적인 의견을 제시하고 3라운드 토론을 완료하는지 정성 평가 |
| 성능/경험 | 평균 응답 시간 | 고민 입력 후 최종 조언 수신까지 15초 이내 |
| 성능/경험 | 페르소나 일관성 | 각 Agent 답변이 부여된 페르소나와 연애관에 부합하는지 정성 평가 |

## MVP 범위

### 포함 범위(Must-Haves)

- 로컬 환경 기반 실행 가능한 데모 구현
- React 프론트엔드 및 FastAPI 백엔드 구동
- 핵심 Agentic Workflow 구현
  - 사용자 고민 분석
  - 6개 Agent의 초기 의견 생성
  - 3라운드 고정 토론 진행
  - 최종 요약 Agent의 결과 도출
- 7개 Agent의 시스템 프롬프트 정의
- `consultation_id`별 in-memory 상담 상태 관리
- SSE 기반 진행 상태 및 Agent 응답 이벤트 전송
- 텍스트 입력 및 결과 출력을 제공하는 기본 웹 UI 구현
- 초기 의견 목록, 토론 로그, 최종 요약 결과의 구조화된 화면 표시

### 제외 범위(Nice-to-Haves)

- 사용자 인증 및 데이터베이스 기반 고민 기록 저장
- 복잡한 시각화 기능
- Agent별 감정선 변화 시각화
- 외부 데이터 검색(RAG) 기능 통합

## 단계별 개발 로드맵

| 단계 | 기간 비중/예상 기간 | 주요 작업 |
| --- | --- | --- |
| 1단계: 환경 설정 및 기초 Agent 정의 | 30% / 4일 | React, FastAPI, LangGraph, LLM 연동 준비. 7개 Agent 시스템 프롬프트 정의. 사용자 입력 분석 및 6개 Agent 초기 의견 생성 기능 구현 |
| 2단계: 핵심 Workflow 및 논의 엔진 구현 | 40% / 6일 | LangGraph 기반 Agent Orchestrator 및 토론 관리 모듈 구현. Agent 간 반박/보완 토론 로직 구현. 토론 로그 기반 최종 요약 Agent 기능 구현. SSE 진행 이벤트 연동 |
| 3단계: 통합 및 데모 완성 | 30% / 4일 | 백엔드와 프론트엔드 통합. 전체 워크플로우 연동 테스트. UI 개선. 성능 및 안정성 테스트. 최종 데모 준비 |

## 기대 효과

1. **기술적 검증**
   - 주관적인 주제에 대해 멀티 Agent 간의 복잡한 추론 및 협업을 통해 최종 결론을 도출하는 Agentic Workflow의 실현 가능성을 검증한다.

2. **서비스적 차별화**
   - 기존 단일 챗봇이 제공하지 못하는 다양한 관점과 논의 과정을 제공하여 재미 요소와 몰입감을 높인다.

3. **사용자 고민 해결 지원**
   - 사용자가 감정적 혼란 속에서 상황을 객관적으로 인지하고, 여러 의견을 비교하여 스스로 최선의 결정을 내릴 수 있도록 돕는다.
