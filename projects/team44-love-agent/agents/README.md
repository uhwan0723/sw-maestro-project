# Agents

LLM 기반 상담 에이전트의 프롬프트, 토론 규칙, 실행 흐름을 관리하는 영역입니다.

## 담당자

| 담당자 | 역할 | 주 담당 폴더 | 담당 범위 |
| --- | --- | --- | --- |
| 임지빈 | 프롬프트 엔지니어링 | `agents/prompts/` | 슈퍼바이저와 6개 에이전트의 역할, 말투, 판단 기준, 라운드별 지시문 |
| 김민우 | 골격 구성 | `agents/workflows/` | 에이전트 호출 순서, 라운드 실행 구조, 데이터 전달 흐름 |
| 신현성 | 프로토콜 | `agents/workflows/` | 메시지 스키마, 라운드별 입출력 규칙, 슈퍼바이저 개입 규칙 |

## 폴더 구조

```text
agents/
├── prompts/                    # 임지빈 주 담당
└── workflows/                  # 김민우, 신현성 공동 담당
```

## 작업 기준

- 임지빈은 프롬프트를 작성할 때 신현성이 정의한 출력 형식과 필수 필드를 지킵니다.
- 김민우는 워크플로우를 구성할 때 프론트엔드가 사용할 수 있는 응답 구조를 함께 고려합니다.
- 신현성은 프로토콜 변경 시 임지빈의 프롬프트와 김민우의 워크플로우에 어떤 영향이 있는지 확인합니다.
- 프롬프트가 출력 형식을 바꾸면 `agents/workflows/` 문서도 함께 갱신합니다.
- 워크플로우가 라운드 순서를 바꾸면 `agents/prompts/`의 라운드 지시문도 함께 점검합니다.

## 필수 합의 항목

- 에이전트 수와 각 에이전트의 역할
- 슈퍼바이저의 개입 시점
- 1라운드, 2라운드, 3라운드의 목적
- 각 라운드의 입력값과 출력값
- 최종 통합 결과에 반드시 포함할 항목
- 프론트엔드로 넘길 데이터 구조
