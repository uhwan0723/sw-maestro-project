## 시나리오 1 — 답장이 2~3일씩 늦어지는 썸 상대

**고민 입력:**
> 썸 타는 상대가 있는데, 카톡 답장이 항상 2~3일씩 늦어요. 보내면 읽음 표시는 바로 뜨는데 답장은 한참 후에 와요. 물어보면 바빠서 그렇다고 하는데, 정말 바쁜 건지 관심이 없는 건지 모르겠어요.

**검증 포인트:** 6개 에이전트 의견이 실제로 다른지

### Supervisor 1단계 출력 예시

```json
{
  "situation_brief": "썸 상대가 읽음 확인 후에도 2~3일씩 답장을 미루는 패턴이 반복되고 있으며, 상대는 바쁨을 이유로 들고 있다.",
  "key_issues": ["연락 패턴이 관심 부족의 신호인지 단순 성격·습관의 차이인지 판단하기 어렵다", "사용자가 직접 확인 요청을 해야 할 시점인지 결정해야 한다"],
  "emotional_state": "읽음 표시를 확인하고도 답장이 오지 않아 불안과 혼란을 느끼고 있으며, 자신의 감정이 과한 것인지 확신하지 못하는 상태다.",
  "debate_goal": "상대의 연락 패턴을 관계 관심도의 지표로 볼 것인지, 아니면 개인 습관으로 수용하고 다른 방식으로 접근할 것인지를 결정한다."
}
```

### Round 1 에이전트 의견 다양성 확인

| 에이전트 | 핵심 입장 (stance) | 제안 행동 |
|---|---|---|
| 현실주의자 | 읽고 며칠씩 답하지 않는 행동은 우선순위에서 밀려 있다는 신호다 | 직접 "답장이 늦을 때 이유가 있으면 알려줘"라고 한 번 명확히 말한다 |
| 공감형 감성론자 | 불안한 마음은 충분히 이해되며, 그 감정 자체가 이 관계에서 원하는 것이 있다는 신호다 | 답장 속도보다 실제 만남의 질을 기준으로 관계를 평가해본다 |
| 신중한 분석가 | 한 가지 패턴만으로 판단하기보다 다른 맥락(약속 이행, 대화 깊이)을 함께 봐야 한다 | 지난 한 달간 만남 횟수·대화 주제 등 다른 관계 지표를 정리해본다 |
| 행동파 조언자 | 추측보다 직접 물어보는 것이 가장 빠른 답이다 | "나 답장 기다리는 게 힘든데, 바쁠 때 짧게라도 알려줄 수 있어?" 라고 요청한다 |
| 균형형 중재자 | 사용자의 불안도 타당하고, 상대방이 습관적으로 연락이 느린 유형일 가능성도 있다 | 한 번만 솔직하게 감정을 전하고 상대의 반응을 지켜본다 |
| 친구형 상담자 | 솔직히 읽고 안 답하는 건 좀 그렇긴 한데, 한 번은 직접 물어봐야 알 수 있어 | 가볍게 "나 답장 기다리면 좀 불안해져" 라고 말해보는 것도 방법이야 |

**검증 결과:**
- [x] 6개 의견의 stance가 모두 상이함 (현실 직시 / 감정 중심 / 맥락 분석 / 즉각 행동 / 절충 / 캐주얼 공감으로 분기)
- [x] 말투가 각 페르소나와 일치함 (현실주의자: 단정적, 친구형: 캐주얼, 공감형: 따뜻)
- [x] JSON 6개 필드 모두 포함

## 시나리오 2 — 전 연인의 갑작스러운 연락

**고민 입력:**
> 3개월 전에 헤어진 전 남자친구가 갑자기 연락이 왔어요. 별 내용은 없고 '잘 지내냐'는 말 한마디였어요. 제가 먼저 정리하자고 했던 이별이었는데, 지금도 미련이 좀 남아 있어서 어떻게 반응해야 할지 모르겠어요.

**검증 포인트:** 슈퍼바이저가 쟁점을 정확히 추출하는지

### Supervisor 1단계 출력 예시

```json
{
  "situation_brief": "사용자가 먼저 이별을 결정했으나 미련이 남은 상황에서 전 연인이 3개월 만에 짧은 안부 연락을 보내왔다.",
  "key_issues": [
    "상대의 연락 의도가 재결합 시도인지 단순 안부인지 불분명하다",
    "사용자 스스로 미련이 남아 있어 반응 선택이 자신의 감정 상태에도 영향을 받는다"
  ],
  "emotional_state": "이별을 주도했음에도 미련이 남아 있어 상대의 연락에 흔들리는 양가감정 상태로 보인다.",
  "debate_goal": "상대의 연락에 응답할 것인지, 응답한다면 어떤 방식으로 할 것인지를 사용자 스스로의 감정 정리와 함께 결정한다."
}
```

### Supervisor 2단계 출력 예시 (Round 1 종합 후)

```json
{
  "agreements": [
    "연락 의도가 불분명한 상황에서 성급하게 재결합을 기대하는 것은 위험하다",
    "사용자가 자신의 감정부터 먼저 정리하는 것이 중요하다"
  ],
  "conflicts": [
    "현실주의자 vs 공감형 감성론자: 상대의 연락을 무시하거나 짧게 응하는 것이 맞는지, 아니면 솔직하게 자신의 감정을 전하며 만남을 제안하는 것이 맞는지",
    "행동파 조언자 vs 신중한 분석가: 지금 바로 연락해 의도를 확인해야 하는지, 아니면 일정 시간을 두고 내 감정을 먼저 정리해야 하는지"
  ],
  "focus_for_round2": "사용자가 자신의 미련과 이별 이유를 충분히 정리한 뒤에 반응하는 것이 나은지, 아니면 지금 솔직하게 대화로 풀어가는 것이 나은지를 집중 논의한다."
}
```

**검증 결과:**
- [x] `key_issues` 2개, 내용이 구체적이고 서로 다른 차원의 쟁점(상대 의도 불명확 / 사용자 감정 혼재)을 다룸
- [x] `debate_goal`이 결론을 내포하지 않는 열린 질문 형태
- [x] `conflicts`에 에이전트 이름이 포함되어 구체적으로 기술됨
- [x] JSON 필드 누락 없음

## 시나리오 3 — 약속을 자꾸 취소하는 연인

**고민 입력:**
> 남자친구가 약속을 자꾸 취소해요. 한 달에 서너 번은 되는 것 같아요. 이유는 거의 설명 안 하고 '미안, 오늘 못 갈 것 같아'라고만 해요. 따지면 예민하다고 하고, 이해하려고 하면 제 마음이 너무 힘들어요.

**검증 포인트:** 최종 요약이 한쪽 편에 치우치지 않는지

### Final Summary 출력 예시

```json
{
  "situation_summary": "남자친구가 한 달에 서너 번씩 약속을 이유 설명 없이 취소하고 있으며, 사용자가 문제를 제기하면 예민하다는 반응이 돌아온다. 사용자는 이해와 감정 표현 사이에서 지속적인 심리적 소진을 겪고 있다.",
  "main_conflict": "현실주의자와 행동파 조언자는 반복 패턴을 문제로 규정하고 명확한 경계 요청을 우선시한 반면, 공감형 감성론자와 균형형 중재자는 상대방의 상황도 고려해 대화 방식 개선을 먼저 시도해야 한다는 입장을 유지했다.",
  "consensus": "현재 상태가 지속될 경우 관계의 신뢰에 손상이 생길 수 있으며, 어떤 방식으로든 사용자의 불편함을 상대방에게 전달하는 것이 필요하다.",
  "final_advice": "약속 취소 자체보다 이유를 공유하지 않는 것과 감정 표현에 '예민하다'는 반응이 누적되는 구조가 핵심 문제입니다. 지금 당장 결정을 내리기보다, 다음에 약속을 잡을 때 '취소할 것 같으면 미리 말해줘'라는 구체적인 요청 한 가지를 시도해보는 것이 현실적인 첫 단계일 수 있습니다. 어떤 선택이 맞는지는 이 시도에 상대가 어떻게 반응하는지를 보면서 스스로 판단하실 수 있을 거예요.",
  "action_steps": [
    "다음 약속 전에 '취소할 것 같으면 하루 전에 알려줘'라고 구체적으로 요청해본다",
    "한 달 동안 취소 횟수와 상황을 간단히 메모해두고, 패턴이 보이면 차분히 대화를 요청한다",
    "현재 자신이 이 관계에서 무엇을 가장 힘들어하는지 짧게 글로 정리해 본인의 감정부터 명확히 파악한다"
  ],
  "caution": "이 답변은 전문 심리상담을 대체하지 않습니다. 관계 내 소통 문제가 지속될 경우 커플 상담 등 전문 도움을 받으시길 권합니다."
}
```

**검증 결과:**
- [x] `final_advice`가 어느 한 에이전트 편을 들지 않고 균형 있게 작성됨
- [x] `action_steps` 3개 포함, 각각 구체적이고 사용자 주어 형태
- [x] `caution`에 면책 문구("이 답변은 전문 심리상담을 대체하지 않습니다") 포함
- [x] `main_conflict`에 에이전트 이름이 구체적으로 기술됨
- [x] JSON 필드 누락 없음
