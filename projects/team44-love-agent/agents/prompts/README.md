# Agent Prompts

슈퍼바이저와 상담 에이전트의 프롬프트를 관리하는 폴더입니다.

## 담당자

- 주 담당: 임지빈
- 프로토콜 검토: 신현성
- 사용자 문구 검토: 박준혁

| 파일 | 설명 | 상태 |
| --- | --- | --- |
| `supervisor.md` | 슈퍼바이저 역할 프롬프트 (질문 해석 → 쟁점 정리 → 최종 통합 지시) | 완료 |
| `relationship_agents.md` | 6개 상담 에이전트 페르소나 프롬프트 | 완료 |

## 에이전트 구성

슈퍼바이저 1개 + 상담 에이전트 6개로 구성됩니다.

| 에이전트 | 역할 관점 |
| --- | --- |
| 슈퍼바이저 | 질문 해석, 라운드 진행, 최종 통합 |
| 공감형 | 감정 중심, 위로와 수용 |
| 분석형 | 상황 분석, 원인 파악 |
| 현실형 | 실현 가능성, 현실적 조언 |
| 도전형 | 관계 재정의, 변화 촉구 |
| 관계형 | 상대방 관점, 관계 유지 |
| 성장형 | 자기 발전, 장기적 관점 |

## 토론 흐름

```text
[슈퍼바이저] 질문 해석 및 토론 목표 설정
    ↓
[1라운드] 6개 에이전트 독립 의견 생성
    ↓
[슈퍼바이저] 쟁점 정리
    ↓
[2라운드] 에이전트 반박 및 보완
    ↓
[3라운드] 에이전트 최종 입장 정리
    ↓
[슈퍼바이저] 최종 답변 통합 → 사용자 전달
```

## 작성 기준

- 각 프롬프트는 역할, 입력값, 출력값, 말투, 금지 사항을 분리해서 작성합니다.
- 출력 형식은 신현성이 정의한 메시지 스키마(`agents/workflows/`)를 따릅니다.
- 사용자에게 직접 보이는 표현은 박준혁의 소통 방식 기준과 충돌하지 않게 작성합니다.
- 에이전트의 성격은 분명하게 나누되, 최종 답변은 슈퍼바이저가 통합할 수 있도록 구조화합니다.
- 프롬프트만 보고도 해당 에이전트가 무엇을 해야 하는지 이해할 수 있어야 합니다.
